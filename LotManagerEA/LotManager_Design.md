# LotManager EA 設計仕様書

## 1. 概要
LotManager EAは、GOLD取引におけるロット管理、ポジションの可視化、およびGUIを用いた高度な分割指値注文（グリッドオーダー）をサポートするMT5用エキスパートアドバイザーです。
裁量トレードの補助ツールとして機能し、リスク管理と資金管理を視覚的かつ直感的に行うことを目的としています。

## 2. 主な機能

### 2.1. 平均取得価格の表示
- 現在チャートに表示されている銘柄（シンボル）の保有ポジション（Buy/Sell別）の平均取得価格を算出し、チャート上に水平線として表示します。
- 新規ポジションの追加や決済が行われるたびに、自動的に計算と水平線の位置が更新されます。

### 2.2. ポジション数と追加可能数の表示
- **現在のオープンポジション数**を表示します。
- **追加可能ポジション数（ロット数）**を口座残高を基準に算出し表示します。
  - 算出基準：口座残高1,000円（または口座通貨）につき0.1ロット。
  - 計算式：`最大許容ロット = (口座残高 / 1000) * 0.1`
  - 追加可能ロット = `最大許容ロット - 現在の保有ロット`
  - 追加可能ポジション数 = `追加可能ロット / 0.1`（0.1ロット固定エントリーの場合）
- Micro口座（1ロット=1,000通貨）とStandard口座（1ロット=10万通貨）で共通の計算基準（残高1000につき0.1ロット）を使用します。

### 2.3. GUIパネルによる分割指値エントリー機能
チャート上に操作パネル（GUI）を表示し、以下の設定をリアルタイムで行えます。

#### 2.3.1. ゾーン分割設定
- 指定した価格帯（例：上限2,900〜下限2,880）を最大5つのゾーンに分割可能。
- 各ゾーン内に等間隔でBuy/Sellの指値注文（Limit/Stop）を設定します（基準ロット：0.1ロット）。

#### 2.3.2. SL/TPの柔軟な設定
以下の3つのモードから選択可能：
1. **全ポジション共通**：すべての待機注文に同じSL/TP価格を設定。
2. **各注文ごと**：個別の注文に対して独立したSL/TPを設定。
3. **ゾーンごと**：ゾーン単位で共通のSL/TPを設定（例：ゾーンAはTP+$20、ゾーンBはTP+$10）。

#### 2.3.3. 直近高値・安値を基準とした自動SL設定
- EAが適用されているチャートの時間足（Timeframe）を自動検出し、その時間足における直近の目立つ高値（Swing High）または安値（Swing Low）を特定します。
- 特定した高値/安値から **$5（500pips）** 離れた位置にSLを自動設定します。
- 「直近」の定義：現在足から過去に遡り、フラクタル（Fractals）または最高値/最安値のロジックを用いて直近の波の頂点/底を判定します。

#### 2.3.4. 想定損益のシミュレーション表示
- 設定した指値注文、SL、TPに基づき、以下のシミュレーション結果をチャート上（またはGUIパネル内）に表示します。
  - **最大想定損失額**：すべての指値注文が約定し、すべてがSLに達した場合の口座通貨ベースの損失額。
  - **最大想定利益額**：すべての指値注文が約定し、すべてがTPに達した場合の口座通貨ベースの利益額。

## 3. インターフェース設計 (GUI)
MT5の標準ライブラリ（Controls）またはカスタム描画関数を使用して、チャート上に以下の要素を持つパネルを作成します。

- **情報表示エリア**:
  - 現在の残高
  - 現在の保有ロット数 / ポジション数
  - 追加可能ロット数 / ポジション数
- **注文設定エリア**:
  - 上限価格・下限価格の入力フィールド
  - ゾーン数（1〜5）の選択ドロップダウン/スライダー
  - 分割数（総注文数）の入力
- **SL/TP設定エリア**:
  - SL/TPモード選択（共通・個別・ゾーン）
  - 直近高値/安値ベースのSL自動計算トグルボタン
- **想定損益エリア**:
  - Total Risk (損失額)
  - Total Reward (利益額)
- **実行ボタン**:
  - 「Place Orders（注文発注）」
  - 「Clear Lines/Orders（クリア）」

## 4. 技術的制約・留意事項
- **GUIの複雑さ**: MT5の標準ダイアログライブラリは柔軟性に欠ける部分があるため、CAppDialog等を利用して実装しますが、コード量が非常に多くなります。
- **直近高安値の判定**: 単純な過去N本ではなく、ZigZagやFractalロジックを内包して「意味のある高値・安値」を動的に見つけるアルゴリズムを実装します。
- **想定損益の計算**: 手数料やスワップは含めず、純粋な価格差×ロット数×ティックバリューで計算します。
