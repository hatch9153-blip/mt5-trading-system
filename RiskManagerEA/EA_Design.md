# MT5 リスク管理EA 仕様書・設計書

**プロジェクト名**: RiskManager EA  
**バージョン**: 1.0  
**作成日**: 2026年3月21日  

---

## 1. システム概要

本ドキュメントは、RiskManager EAの詳細仕様と内部設計を記述する。EAはMQL5で実装され、MT5の「エキスパートアドバイザー」として動作する。エントリー判断はユーザーが行い、EAは以下の3つのリスク管理機能を提供する。

1. **時間帯フィルター**: 優位性のある時間帯のみエントリーを許可
2. **利益目標達成終了**: 1日+20%達成でトレードを終了
3. **連敗ストッパー**: 連敗後のエントリーを自動ブロック

---

## 2. 入力パラメーター仕様

| パラメーター名 | 型 | デフォルト値 | 説明 |
|---------------|-----|-------------|------|
| `DailyProfitTarget` | double | 20.0 | 1日の利益目標（開始残高に対する%） |
| `MaxConsecutiveLosses` | int | 3 | 通常時間帯の連敗ストッパー発動回数 |
| `NYMaxConsecutiveLosses` | int | 1 | NY時間帯の連敗ストッパー発動回数 |
| `EnableTimeFilter` | bool | true | 時間帯フィルターの有効/無効 |
| `EnableProfitTarget` | bool | true | 利益目標達成終了の有効/無効 |
| `EnableStreakStopper` | bool | true | 連敗ストッパーの有効/無効 |
| `AlertOnBlock` | bool | true | ブロック発動時のアラート通知 |

---

## 3. 時間帯フィルター仕様

### 3.1 許可時間帯（JST）

```
08:00 - 09:00  東京時間開始
10:00 - 10:30  東京時間中盤
15:00 - 16:00  東京時間終盤
21:30 - 00:00  NY時間（翌日0時まで）
```

### 3.2 時間判定ロジック

MT5のサーバー時間（UTC）をJST（UTC+9）に変換して判定する。

```
JST = ServerTime + 9時間
```

### 3.3 ブロック時の動作

- `OnTick()` 内で新規注文イベントを監視
- 許可時間帯外での注文を検知した場合、`OrderSend` をブロック
- チャート上部に「⛔ 時間帯フィルター: 現在エントリー禁止時間帯です」を表示

---

## 4. 利益目標達成終了仕様

### 4.1 開始残高の記録

- `OnInit()` 実行時に `AccountInfoDouble(ACCOUNT_BALANCE)` を取得し、グローバル変数に保存
- 日付が変わった場合（`TimeDay` の変化を検知）、残高を再取得してリセット

### 4.2 目標達成判定

```
現在残高 >= 開始残高 × (1 + DailyProfitTarget / 100)
```

### 4.3 達成時の動作

- その日の新規エントリーをブロック
- アラート: 「✅ 本日の利益目標達成！新規エントリーをブロックしました」
- チャート上部に達成ステータスを表示

---

## 5. 連敗ストッパー仕様

### 5.1 連敗カウントロジック

- `OnTradeTransaction()` で決済イベントを監視
- 直近の決済済みトレードの損益（`DEAL_PROFIT`）を確認
- 損失（`DEAL_PROFIT < 0`）の場合、連敗カウントをインクリメント
- 利益（`DEAL_PROFIT >= 0`）の場合、連敗カウントをリセット

### 5.2 発動条件

| 時間帯 | 発動条件 |
|--------|---------|
| 通常時間帯（08:00〜21:30） | 連敗カウント >= `MaxConsecutiveLosses`（デフォルト3） |
| NY時間帯（21:30〜翌00:00） | 連敗カウント >= `NYMaxConsecutiveLosses`（デフォルト1） |

### 5.3 発動時の動作

- その日の新規エントリーをブロック
- アラート: 「⛔ 連敗ストッパー発動: 本日の新規エントリーをブロックしました」
- チャート上部に連敗数と発動ステータスを表示

### 5.4 リセット条件

- 日付が変わった時点（翌日08:00ではなく、日付変更時）に連敗カウントとブロック状態をリセット

## 6. チャート表示仕様

### 6.1 ステータスパネル

チャート左上に以下の情報を常時表示する。

```
[RiskManager v1.0]
開始残高: 12,345.67 USD
現在残高: 13,210.00 USD
利益率:   +7.0%  (目標: +20.0%)
連敗数:   1 / 3
時間帯:   ✅ エントリー許可 (東京時間)
本日状態: 稼働中
```

### 6.2 ブロック状態の表示

ブロック発動時はステータスパネルの「本日状態」欄を変更する。

| ブロック原因 | 表示 |
|-------------|------|
| 時間帯フィルター | ⛔ 禁止時間帯 |
| 利益目標達成 | ✅ 目標達成・終了 |
| 連敗ストッパー | ⛔ 連敗ストッパー発動 |

## 7. 内部設計

### 7.1 グローバル変数

| 変数名 | 型 | 説明 |
|--------|-----|------|
| `g_startBalance` | double | 本日の開始残高 |
| `g_targetBalance` | double | 本日の利益目標残高 |
| `g_consecutiveLosses` | int | 現在の連敗カウント |
| `g_isBlockedByTime` | bool | 時間帯フィルターによるブロック状態 |
| `g_isBlockedByProfit` | bool | 利益目標達成によるブロック状態 |
| `g_isBlockedByStreak` | bool | 連敗ストッパーによるブロック状態 |
| `g_lastTradeDay` | int | 最後にリセットした日付 |
| `g_lastDealTicket` | ulong | 最後に処理したDealのチケット番号 |

### 7.2 主要関数

| 関数名 | 説明 |
|--------|------|
| `OnInit()` | 初期化処理。開始残高の記録、グローバル変数の初期化 |
| `OnTick()` | 毎ティック実行。日付変更チェック、ブロック状態の更新、チャート表示の更新 |
| `OnTradeTransaction()` | 取引イベント処理。決済検知と連敗カウントの更新 |
| `IsAllowedTime()` | 現在時刻が許可時間帯かどうかを判定 |
| `CheckProfitTarget()` | 利益目標達成チェック |
| `UpdateChartDisplay()` | チャート上のステータスパネルを更新 |
| `ResetDailyState()` | 日次リセット処理 |
| `OnDeinit()` | 終了処理。チャートオブジェクトの削除 |

### 7.3 注文ブロックの実装方針

MT5のEAは `OnTick()` 内で注文を監視するが、ユーザーの手動注文を直接インターセプトすることはできない。そのため、以下の方式でブロックを実装する。

1. **視覚的警告**: チャートに大きく「エントリー禁止」を表示し、ユーザーに通知する
2. **アラート通知**: MT5のアラートダイアログを表示する
3. **自動クローズ（オプション）**: ブロック状態中に新規ポジションが開かれた場合、即座にクローズする

> **注意**: MQL5の仕様上、ユーザーの手動注文を完全にブロックすることは技術的に困難なため、本EAは主に「視覚的・音声的警告」によってユーザーの行動を抑制する方式を採用する。

## 8. テスト計画

| テストケース | 期待結果 |
|-------------|---------|
| 許可時間帯内でのエントリー | ブロックなし、通常動作 |
| 禁止時間帯でのエントリー試行 | アラート表示、警告メッセージ |
| 開始残高の+20%達成 | ブロック発動、アラート表示 |
| 通常時間帯で3連敗 | ブロック発動、アラート表示 |
| NY時間帯で1連敗 | ブロック発動、アラート表示 |
| 日付変更後 | 全ブロック状態・連敗カウントのリセット |
| EA削除後 | チャートオブジェクトがすべて削除される |
