# MT5 リスク管理EA 要件定義書

**プロジェクト名**: RiskManager EA  
**バージョン**: 1.0  
**作成日**: 2026年3月21日  
**対象プラットフォーム**: MetaTrader 5（MT5）  

---

## 1. プロジェクトの背景と目的

### 1.1 背景

口座保有者（以下、ユーザー）は、GOLDmicroを主要銘柄として、上位足のトレンド方向に沿った下位足の反転タイミングを狙う裁量トレードを行っている。取引データ（11,004件）の分析により、以下の課題が明確になった。

| 課題 | データの根拠 |
|------|-------------|
| 東京時間は非常に高い勝率（約80%）を誇るが、欧州・NY時間に入ると成績が急激に悪化する | セッション別分析で東京勝率73.7%、NY勝率29.6% |
| 3連敗後の次トレードの勝率が7.3%に激減する「規律崩壊パターン」が存在する | 連敗パターン分析より |
| 利益目標達成後もトレードを継続し、オーバートレードに陥る | 1日200回超でP/Lがマイナスに転じる |
| NY時間の一方向トレンドに対して逆張りナンピンを繰り返し、口座に致命傷を与える | 最大損失日（3/13）分析より |

### 1.2 目的

本EAは、ユーザーの裁量トレードを「補助・監視」するリスク管理専用EAである。エントリー判断はユーザーが行い、EAは「やってはいけないこと」を物理的にブロックすることで、感情を排除した機械的なリスク管理を実現する。

**最終目標**: 1日の開始残高に対して+20%の利益を機械的に積み重ね、20営業日で安定した複利成長を達成する。

## 2. システム要件

### 2.1 機能要件

#### FR-01: 時間帯フィルター

| 要件ID | 要件内容 |
|--------|---------|
| FR-01-1 | 以下の許可時間帯のみ新規エントリーを許可する（JST基準） |
| FR-01-2 | 許可時間帯以外は新規注文をブロックし、アラートを表示する |

**許可時間帯（JST）:**

| 時間帯 | 理由 |
|--------|------|
| 08:00〜09:00 | 東京時間開始・動きやすい |
| 10:00〜10:30 | 東京時間中盤・優位性が高い |
| 15:00〜16:00 | 東京時間終盤・動きやすい |
| 21:30〜翌00:00 | NY時間・高ボラティリティ（制限付き） |

#### FR-02: 1日の利益目標達成によるトレード終了

| 要件ID | 要件内容 |
|--------|---------|
| FR-02-1 | EA起動時の口座残高を「本日の開始残高」として記録する |
| FR-02-2 | 現在の口座残高が開始残高の120%（+20%）以上になった時点で、その日の新規エントリーをブロックする |
| FR-02-3 | 目標達成時にアラートを表示し、ユーザーに通知する |
| FR-02-4 | 翌日のEA起動時（または翌日08:00）に自動的にリセットする |

#### FR-03: 連敗ストッパー

| 要件ID | 要件内容 |
|--------|---------|
| FR-03-1 | 通常時間帯（08:00〜21:30）において、3連敗した時点でその日の新規エントリーをブロックする |
| FR-03-2 | NY時間帯（21:30〜翌00:00）において、1連敗した時点でその日のNY時間のエントリーをブロックする |
| FR-03-3 | 連敗カウントは決済済みトレードのみを対象とする（含み損は対象外） |
| FR-03-4 | ブロック発動時にアラートを表示し、ユーザーに通知する |
| FR-03-5 | 翌日のEA起動時（または翌日08:00）に自動的にリセットする |

### 2.2 非機能要件

| 要件ID | 要件内容 |
|--------|---------|
| NFR-01 | MT5のすべてのバージョンで動作すること |
| NFR-02 | 複数の通貨ペア・銘柄のチャートに同時に適用可能であること |
| NFR-03 | パラメーターはEAのプロパティ画面から変更可能であること |
| NFR-04 | EAを削除した際に、チャートに影響を与えないこと |

---

## 3. 制約条件

- 本EAはエントリーの補助・監視のみを行い、自動売買（自動エントリー）は行わない
- 時間帯はJST（日本標準時、UTC+9）を基準とする
- ブロック中でも既存ポジションの手動決済は妨げない

---

## 4. 用語定義

| 用語 | 定義 |
|------|------|
| 連敗 | 直近の決済済みトレードが連続して損失（P/L < 0）であった回数 |
| 開始残高 | EA起動時点の口座残高（Balance） |
| 利益目標 | 開始残高 × 1.20（+20%） |
| 許可時間帯 | 新規エントリーが許可される時間帯 |
| ブロック | 新規注文の送信を禁止する状態 |
