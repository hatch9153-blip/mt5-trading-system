# MT5 Trading System プロジェクトコンテキスト

## プロジェクト概要

本プロジェクトは、MT5（MetaTrader 5）での裁量トレードを補助し、リスク管理を機械的に行うためのシステム群です。過去の取引データ（約11,000件）の詳細なデータ分析に基づき、ユーザーのトレードスタイルの強み（東京時間のレンジ相場での分割エントリー）を活かしつつ、弱点（欧州・NY時間での一方向トレンドに対する無限ナンピン、利益目標達成後のオーバートレード）を物理的に防ぐことを目的としています。

また、新たにSMC（Smart Money Concepts）戦略に基づく完全自動売買EA（SMCMasterEA）の開発を進めています。

---

## 開発の経緯

1. **データ分析**: XMTradingの取引履歴HTMLレポートをPythonで解析し、勝率・損益分布・時間帯別の成績を可視化しました。
2. **課題の特定**: 東京時間（JST 08:00〜16:00）は非常に高い勝率（約80%）を誇る一方、NY時間（JST 21:30以降）の逆張りナンピンが口座に致命傷を与えていることが判明しました。また、3連敗した後の次トレードの勝率が7.3%に激減するという「規律崩壊パターン」がデータで証明されました。
3. **インジケーター開発**: トレンドの方向性と反転タイミングを視覚的に捉えるため、2つのストキャスティクス（9,3,3 と 60,3,3）を組み合わせたサブウィンドウ背景色変更インジケーターを開発しました。
4. **リスク管理EA開発**: 分析結果から導き出された「やってはいけないこと」をシステム的にブロックするEAを開発しました。
5. **SMC自動売買EA開発**: SMC（Smart Money Concepts）の各要素を統合した完全自動売買EAの設計・開発を開始しました。

## ディレクトリ構成と成果物

### `/SMCMasterEA/`

SMC（Smart Money Concepts）戦略を用いた完全自動売買EA。XMTrading円建てMicro口座（1lot=1,000通貨）向けに設計。

**対象銘柄（10銘柄）**: GOLD/XAUUSD、USDJPY、GBPJPY、EURJPY、GBPUSD、EURUSD、AUDJPY、AUDUSD、GBPAUD、EURAUD

**実装するSMC要素（9要素）**:

| 要素 | 概要 |
|---|---|
| Order Block (OB) | 機関投資家の注文集中帯（大陽線/大陰線の起点）の検出 |
| Fair Value Gap (FVG) | 3本のローソク足間で生じる価格のインバランス検出 |
| BOS / CHoCH | トレンド継続（BOS）および転換（CHoCH）シグナルの検出 |
| Liquidity Sweep | 直近高安値を抜けた後の反転（ストップハント）の検出 |
| Premium / Discount Zone | スイングのフィボナッチ50%ラインによる割高・割安ゾーン判定 |
| Judas Swing | ロンドン/NYオープン直後の騙しの動き（逆行）の検出 |
| Silver Bullet | 特定時間帯（NY 10:00-11:00 EST等）の高勝率セットアップ |
| Weekly PO3 | 週間の値動きサイクル（Accumulation→Manipulation→Distribution）の検出 |
| SMT Divergence | 相関銘柄間での高値・安値のダイバージェンス検出 |

**リスク管理ルール**:

| 項目 | 設定値 |
|---|---|
| ロット計算 | 残高1,000円につき0.1lot（最大1.0lot / 残高10,000円上限） |
| 日次利益制限 | 開始残高比+20〜25%達成でその日のトレード停止 |
| Stop Loss | 直近高安値から±5ポイント |
| Take Profit | リスクリワード1:2以上 または次の流動性ターゲット |
| 時間足 | M1〜H4のマルチタイムフレームでコンフルエンス判定 |

**ファイル一覧**:
- `EA_Requirements.md`: EA要件定義書
- `EA_Design.md`: EA設計書（各SMC要素の検出ロジック・エントリー条件・ロット計算等）
- `SMCMasterEA.mq5`: EA本体ソースコード（実装予定）

### `/RiskManagerEA/`
裁量トレードの感情的な判断を排除し、事前に定めたリスク管理ルールを強制するEA（エキスパートアドバイザー）。

- `RiskManager.mq5`: EA本体のソースコード。以下の3つの主要機能を持ちます。
  1. **時間帯フィルター**: 優位性のある時間帯（08:00-09:00, 10:00-10:30, 15:00-16:00, 21:30-00:00）のみエントリーを許可。
  2. **利益目標達成終了**: 1日の開始残高に対して+20%の利益を達成した時点で、その日の新規エントリーをブロック。
  3. **連敗ストッパー**: 通常時間は3連敗、ボラティリティの高いNY時間は1連敗でその日の新規エントリーをブロック。
- `EA_Requirements.md`: EAの要件定義書。
- `EA_Design.md`: EAの仕様書・設計書。

---

### `/Indicators/`

トレードの補助となるカスタムインジケーター。

- `StochasticZone.mq5`: サブウィンドウに短期・長期のストキャスティクスを表示し、長期（60,3,3）の値と向きに応じてサブウィンドウの背景色（矩形オブジェクト）をゾーンごとに変更するインジケーター。
- `StoTest.mq5`: 動作確認用のシンプルなストキャスティクスインジケーター。

---

### `/AnalysisReports/`

取引履歴データの分析結果をまとめたMarkdownレポートおよびグラフ画像。

- `mt5_analysis_report.md`: 全体的な取引成績・損益・統計情報の基礎分析レポート。
- `trade_improvement_proposal.md`: 基礎分析に基づく4つの改善提案レポート。
- `trade_style_analysis.md`: 連勝・連敗パターン、ナンピン傾向、時間帯別の詳細分析レポート。
- `trade_deep_analysis.md`: 好調期の成功要因と、大敗日の崩壊パターンの深掘り分析レポート。
- `entry_pattern_analysis.md`: 東京時間における分割エントリーグループの抽出と最適解の分析レポート。
- `march13_reverse_analysis.md`: 最大損失日（3月13日）において、もし順張りでエントリーしていた場合の最大想定利益を算出したシミュレーションレポート。
- `*.png`: 各種分析結果を可視化したグラフ画像群。

## 今後の展望

現在のシステムは「リスク管理の強制（守り）」と「SMC自動売買（攻め）」の両輪で構成されています。今後のステップとして、以下の機能拡張が考えられます。

- **SMCMasterEA本体の実装**: 設計書に基づくMQL5コードの実装・テスト・最適化。
- **ブレイクイーブン機能**: 含み益が一定以上になったポジションの2〜3割を、建値にストップロスを移動して放置（利を伸ばす）する機能。
- **全体含み損による強制決済**: 口座残高の一定割合（例：5%）の含み損を抱えた時点で全ポジションを強制決済する究極のセーフティネット。
- **バックテスト・最適化**: MT5ストラテジーテスターを用いたSMCMasterEAのパラメーター最適化。
